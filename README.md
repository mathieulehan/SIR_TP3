## Quelles sont les limites d’une base données orientées document ?
```
NoSQL nous permet de stocker ce que l'on veut dans les documents (adéquat pour les applications web & mobile)
, mais c'est aussi ce qui devient sa limite :
il est plus facile de faire des erreurs, la duplication de données est possible
ce qui implique que leur mise à jour devient plus lente ;
```
```
En somme il y a moins de cohérence que dans une base de données
relationnelle encadrée par des règles (clés étrangères).
```
```
Il n'y a pas ailleurs pas de JOIN en NoSQL comme on peut le trouver en SQL
(même si certains SGBD permettent de faire des choses similaires).
```
